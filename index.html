Midterm Proposal

Isabel Chai & Crystal Wu

For the midterm project, we plan to create a Tetris game, a classic puzzle video game, using p5.js. In the beginning, there should be a start interface that prompts the user with different game difficulty levels (determined by the drop speed) and a start button. After the user makes a selection and starts the game, the game interface should be on the left, and on the right is the scoring page. The user should be able to rotate the shape and move tetrominoes using different keys. If squares fill up the entire row, it should clear the row and move other rows down a level. If all the rows fill up to the top, then the game will end and report the score to the user. The game will trigger sound when the tetrominoes drop at a certain point and when the row is clear.

From technology's perspective, we need to figure out the mechanism to detect if the row is filled or not. If the row is filled up, how to clear the row and move other rows down a level. It might involve some collision detection for us to determine this. Also, if we are using images for tetromino, we are not sure if image collision detection is similar to what we did before.
